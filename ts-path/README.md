# ts-path — Full-Stack TypeScript (React + Express) in 20 hours

A fast, 80/20 path to building full-stack web apps with **TypeScript + React** (frontend) and **Express + TS** (backend). 10 sessions × 2 hours.

## Calibration (important)
I already know: **Express, Prisma, JWT auth, TypeScript basics** (built a typed habit-tracker API — see `../phase_2`). So this path is **React-weighted**. Express/TS sessions move fast; React and full-stack wiring get the depth.

**What 20h gets me:** a working full-stack app + a real React foundation. NOT mastery — performance tuning, advanced patterns, testing come after.

## How each session works
1. **Teach** — the concept, explained as simply as possible (bullets + diagram + example).
2. **Explain-back (Socratic loop)** — I explain it in my own words → mentor points out mistakes → re-teaches what I missed → repeat until I can explain it solo.
3. **Build** — add a piece to the app (learning by doing).
4. **15-min review** at the end.
5. **1-page summary** written to `sessions/` — bullets/diagram/example, reviewable in 5 min.
6. **10-question quiz** — increasing difficulty; each answer scored, mistakes explained. Total score /10 logged.

## The app we build
Chosen at kickoff (see `ROADMAP.md`). Starts as a React app hitting a public API (instant results), then becomes full-stack against our own Express+TS backend with auth.

## Files
- `ROADMAP.md` — the 10 sessions + resources.
- `MENTOR.md` — the exact teaching method (feed to any fresh Claude to continue).
- `sessions/` — per-session 1-page summaries + quiz scores.
- `app/` — the code we build.

## Resume on any machine
> Read `ts-path/MENTOR.md` + `ROADMAP.md` + the latest `sessions/` file, and continue from the next session.
